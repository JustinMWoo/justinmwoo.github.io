iMSTK Plugin for Unreal Engine


Overview

This plugin provides access to iMSTK's medically accurate soft body simulation within Unreal Engine by replacing the existing physics simulation for selected objects. Components and actors from the plugin can be added to quickly and easily set up the simulation in any level. The plugin also allows for the importing of tetrahedral meshes and image data into the engine, which can then be used with the provided components to simulate volumetric tissues and other materials.

Tetrahedral Meshes and Image Data

Tetrahedral meshes can be imported into Unreal, saving the volumetric mesh as well as extracting a surface mesh and saving them as assets. When these file types get imported into the editor, the data is passed into iMSTK to extract the vertex positions and indices for connecting the tetrahedra. These values are saved onto a TetrahedralMeshAsset which can be used in conjunction with components provided in the plugin to simulate volumetric models. Similarly, image data files can also be imported and used in Unreal Engine.

PBD Models

Position based dynamics models can be simulated by adding a PBDModel component to an actor. Several preset geometry types can be selected, including capsules, cylinders and spheres. A surface mesh geometry can also be selected to take the relevant information from a static mesh component on the actor and import it into iMSTK at runtime by extracting the position vertex buffer and index buffer from the static mesh and creating the same mesh in the simulation within iMSTK. The vertices can be fixed to their starting position in iMSTK by either listing specific vertices or by adding a separate actor which will freeze any vertices inside of its bounds. Variables pertaining to the constraints that can be applied to a model with the selected settings will appear dynamically. These constraints will be applied to the model within iMSTK.
At the start of simulation the static mesh component on the actor is replaced with a procedural mesh component and the positions of each vertex is then updated by the iMSTK subsystem.

Interactions

Interactions are defined on a model-to-model basis for each simulation. Models can be set to collide by assigning the interaction between them, which can be further customized by adjusting values such as friction, stiffness and restitution depending on the type of models in the collision. With a controller actor, you may also select a multitude of other interaction types including cutting, burning and grasping.

Controllers

Controllers can be added to a level to allow the player to quickly create a tool to interact with the simulation. Controllers can be set to have the same geometry types as a PBD models, but have additional settings for how they interact with other models and can be controlled directly. The controls for the model can be set on the blueprint as well as whether the tool should be directly moved to the position or moved using forces, allowing it to be impacted by opposing forces as well. The forces to move the object to the desired location are calculated in Unreal and the value is passed into iMSTK. Then the actual location of the model is relayed back to Unreal to display the position to the player.
